# Secure Files Compressor (SFC.exe) 项目说明

---

## 一、项目简介

**SFC.exe** 是一个基于 **Huffman 编码与 AES 加密**实现的文件归档工具，支持对**多个文件与目录进行压缩与解压缩**。

该项目主要用于研究**文件系统归档结构、数据压缩以及基础加密实现**。

目前程序仅提供**命令行交互（CLI）**，尚未实现图形界面（GUI）。

---

## 二、功能特性

- 支持**多个文件与目录的归档压缩与解压缩**
- 使用 **Huffman 编码**实现数据压缩
- 支持 **AES-128 加密**与 **SHA-256** 密钥哈希
- 采用 **分块压缩/解压机制**（8MB分块），降低内存占用
- 引入 **逻辑根目录（Logical Root）结构**以统一归档路径
- 实现 **路径重复跳过机制**以避免部分重复文件处理

## 五、压缩过程详解

### 5.1 整体流程

压缩过程主要分为以下几个阶段：

```
用户指定文件/目录
    │
    ▼
写入文件头 (HeaderWriter)
    │
    ▼
扫描目录结构 (EntryProcessor + BinaryStandardWriter)
    │
    ▼
分块读取源文件 (DataLoader)
    │
    ▼
Huffman编码压缩 (Heffman)
    │
    ▼
AES加密 (Aes)
    │
    ▼
写入数据块 (DataExporter)
    │
    ▼
加密目录块 (BinaryStandardLoader)
    │
    ▼
生成.sy文件
```

### 5.2 目录扫描 (EntryProcessor)

程序使用 **BFS（广度优先搜索）** 算法遍历目录树：

1. 首先写入**逻辑根节点**，作为所有压缩项的顶层容器
2. 处理用户指定的原始文件/目录列表
3. 对每个目录，将其加入队列
4. 循环从队列取出目录，处理其内容
5. 如果遇到子目录，将其加入队列继续遍历

### 5.3 二进制编码 (BinaryStandardWriter)

目录结构被转换为二进制格式，按顺序写入：

- **文件条目**: 标志 + 文件名长度 + 文件名 + 原始大小 + 压缩后大小（预留）
- **目录条目**: 标志 + 目录名长度 + 目录名 + 子元素数量
- **符号链接**: 标志 + 链接名长度 + 目标路径长度 + 链接名 + 目标路径
- **逻辑根**: 标志 + 根名称长度 + 根名称 + 子元素数量

### 5.4 分块处理

为了控制内存使用，程序采用**分块处理策略**：

- 每处理 **16KB** 的目录数据，写入一个**分割标准**
- 分割标准记录了每个数据块的大小，供解压时读取
- 文件数据按 **8MB** 分块压缩，每块独立生成Huffman树

### 5.5 Huffman压缩与加密

每个8MB数据块的压缩流程：

1. 统计块中各字节的频率
2. 构建Huffman树
3. 生成各字节的编码表
4. 序列化Huffman树（用于解压时恢复树）
5. 对序列化后的树进行AES加密并写入
6. 对原始数据进行Huffman编码
7. 对编码后的数据进行AES加密并写入

## 六、解压过程详解

### 6.1 整体流程

```
读取.sy文件
    │
    ▼
读取文件头，验证魔数
    │
    ▼
读取目录块，解密后解析 (BinaryStandardLoader + EntryParser)
    │
    ▼
创建目录结构
    │
    ▼
读取数据块
    │
    ▼
AES解密
    │
    ▼
Huffman解码 (Heffman)
    │
    ▼
写入解压数据 (DataExporter)
    │
    ▼
恢复原始文件/目录
```

### 6.2 目录块读取

1. 首先读取文件头，验证魔数和版本
2. 根据目录偏移量定位到目录块
3. 逐块读取分割标准
4. 如果数据被加密（通过IV标志判断），先进行AES解密
5. 解析解密后的二进制数据，提取文件/目录信息
6. 将解析结果加入队列，供后续创建文件使用

### 6.3 数据解压

1. 读取加密的Huffman树数据
2. AES解密树数据
3. 反序列化恢复Huffman树
4. 读取加密的压缩数据
5. AES解密
6. Huffman解码得到原始数据
7. 写入输出文件
8. 重复直到该文件所有数据块处理完毕

## 七、Huffman压缩算法

### 7.1 算法概述

Huffman编码是一种基于字符频率的无损压缩算法：
- 频率高的字符使用较短的编码
- 频率低的字符使用较长的编码
- 编码具有唯一前缀，不会产生歧义

### 7.2 实现要点

**频率统计**：
- 统计待压缩数据中每个字节(0-255)出现的次数
- 程序支持多线程分别统计，最后合并

**树构建**：
- 将每个字符作为叶子节点
- 每次选取频率最小的两个节点合并
- 重复直到只剩一个根节点
- 特殊情况：只有一个字符时，直接编码为"0"

**编码生成**：
- 从根到叶子的路径：左子树=0，右子树=1
- 路径即为该字符的编码

**特殊情况处理**：
- 单字符输入：根节点本身就是叶子
- 每个8MB数据块使用独立的Huffman树

### 7.3 树序列化

压缩时需要将Huffman树写入文件，供解压时恢复树：
- 使用标记区分内部节点('r')和叶子节点('l')
- 按前序遍历顺序记录每个节点
- 叶子节点后跟字符值

## 八、AES加密算法

### 8.1 算法概述

AES (Advanced Encryption Standard) 是美国政府采用的对称加密标准。

本程序实现的是 **AES-128**：
- 密钥长度：128位（16字节）
- 加密轮数：10轮
- 分组大小：128位（16字节）

### 8.2 实现要点

**密钥处理**：
- 用户输入任意长度的密钥
- 使用SHA-256哈希，取前16字节作为实际密钥
- 通过密钥扩展生成44个轮密钥

**加密步骤**（每轮）：
1. **SubBytes**: 字节替换（S盒非线性变换）
2. **ShiftRows**: 行移位（破坏列相关性）
3. **MixColumns**: 列混淆（GF(2^8)域线性变换）
4. **AddRoundKey**: 与轮密钥异或

最后一轮不执行MixColumns。

**解密步骤**是加密的逆过程。

### 8.3 加密模式

程序采用**ECB模式**（简化实现）：
- IV初始化为全0
- 每个数据块独立加密
- 块结构：分割标志 + 块大小 + IV + 加密数据

## 九、.sy 文件格式

### 9.1 整体结构

```
┌─────────────────────────────────┐
│         文件头 (11字节)          │
│  魔数 | 策略 | 版本 | 偏移量    │
├─────────────────────────────────┤
│         目录块 (二进制编码)       │
│  分割标准 | 逻辑根 | 文件条目    │
│  目录条目 | ... | 分割标准      │
├─────────────────────────────────┤
│          魔数 (4字节)           │
├─────────────────────────────────┤
│         数据块 (加密数据)        │
│  加密树 | 加密数据 | 加密树 ... │
├─────────────────────────────────┤
│          魔数 (4字节)           │
└─────────────────────────────────┘
```

### 9.2 目录条目格式

| 类型 | 格式 |
|------|------|
| 文件 | 标志 + 文件名长度 + 文件名 + 原始大小 + 压缩大小 |
| 目录 | 标志 + 目录名长度 + 目录名 + 子元素数 |
| 符号链接 | 标志 + 链接名长度 + 目标路径长度 + 链接名 + 目标路径 |
| 逻辑根 | 标志 + 根名称长度 + 根名称 + 子元素数 |

### 9.3 分割标准

每个数据块前有一个分割标准（25字节）：
- 分割标志 ('2')
- 块大小（8字节）
- IV值（16字节）

## 十、编译与运行

### 10.1 编译要求

- **C++ 标准**: C++20
- **优化等级**: O3（禁止LTO）
- **依赖**: OpenSSL (SHA256, RAND)

### 10.2 使用说明

```bash
# 压缩模式
1. 选择操作模式: 1
2. 输入要压缩的文件/文件夹路径
3. 输入 'done' 完成添加
4. 指定输出目录
5. 输入输出文件名
6. 输入加密密钥

# 解压模式
1. 选择操作模式: 2
2. 输入 .sy 文件路径
3. 指定输出目录
4. 输入解密密钥
```
